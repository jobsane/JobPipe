# Roadmap

## Now

- Establish JobPipe as the clean public baseline
- Remove stale worknames and old public framing
- Keep the repo candidate-first, hiring-aware, local-first, and privacy-respecting
- Tighten the repo around one coherent OSS-first story
- Improve trust, traceability, and follow-up quality in the current workflow

## Next

- Strengthen explicit decision support:
  - job claims
  - hiring-aware decision tables
  - candidate evidence units
  - candidate narrative profiles
  - watchlists and change events
- Improve runtime/data boundaries and local state handling
- Make the public framework/toolkit surface easier to understand and extend

## Later

- Calibration and outcome-aware learning
- Cleaner OSS/private seam
- More reusable contribution surfaces and examples
- Optional team or advisor workflows built on top of the same core substrate

## Out Of Scope For Now

- full recruiter product
- ATS feature set
- mass auto-apply
- broad workflow automation
- generic AI copilot branding
- premature repo split into public/private implementations
