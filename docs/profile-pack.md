# Profile Pack

Jobpipe is built around a `profile_pack.md` file.

This file acts as the main truth source for what the candidate is targeting and how the workflow should interpret relevance.

## Start here

```powershell
copy profile_pack.example.md profile_pack.md
```

Then edit `profile_pack.md` to reflect your own search.

## Typical content

A profile pack can include:
- target role titles
- preferred seniority
- geography or postal preferences
- hard-no role types
- positive role signals
- domain signals
- noise signals
- career evidence in concise bullets

## Why this matters

The better the profile pack reflects the real search, the better Jobpipe can:
- filter noise
- prioritize correctly
- score with more consistency
- produce more useful outputs

## Practical advice

Keep it honest and specific.

Do not try to make the profile pack say everything. Make it say the most important things clearly.
