## Scope Rules

Keep contributions aligned with the current thesis:

- candidate-first
- hiring-aware
- local-first
- privacy-respecting
- evidence-backed
- decision-oriented

Do not broaden the repo toward:

- recruiter tooling
- ATS workflows
- generic AI copilot surfaces
- broad automation for its own sake

## Naming Rules

- Use `JobPipe` consistently.
- Do not reintroduce retired worknames in docs, metadata, comments, scripts, or public messaging.

## Practical Rules

- Keep changes small and reviewable.
- Preserve the current working code unless there is a clear reason to refactor it.
- Prefer boring, maintained dependencies.
- Keep sensitive user data and generated runtime state out of version control.
