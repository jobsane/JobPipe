# Contributing

Thanks for your interest in Jobpipe.

This project is currently built and maintained as a solo-led project, but suggestions, issues, and focused contributions are welcome.

## Before you start

Please keep in mind what Jobpipe is trying to be:

- practical
- structured
- transparent
- lightweight
- useful in a real workflow

This is not a repo for broad, speculative feature sprawl. Small, focused improvements are preferred.

## Good contribution areas

- bug fixes
- setup clarity
- documentation improvements
- dashboard and report usability
- better error handling
- workflow improvements that keep scope under control

## Before opening a PR

Please:
- open an issue first for larger changes
- explain the problem being solved
- keep the change small and focused
- avoid mixing refactors with feature work
- update documentation when behavior changes

## Pull request expectations

A good PR should explain:
- what changed
- why it changed
- risks or trade-offs
- how it was tested

## Style expectations

- prefer small diffs
- keep logic readable
- avoid unnecessary dependencies
- preserve traceability and output clarity
- do not widen scope without a clear reason

## Questions and ideas

The best way to contribute right now is through:
- issues
- bug reports
- focused suggestions
- documentation fixes
- usability feedback

## Documentation rule

If a change affects behavior, terminology, or architecture, update the relevant docs in the same change. The repo should not carry competing explanations of the same system.

Thanks for helping make Jobpipe clearer, safer, and more useful.
